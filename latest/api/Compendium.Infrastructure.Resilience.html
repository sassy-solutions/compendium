Namespace Compendium.Infrastructure.Resilience

Classes

CircuitBreaker

Default implementation of a circuit breaker that tracks failures and manages state transitions. Provides protection against cascading failures by temporarily blocking requests to failing services.

CircuitBreakerOptions

Configuration options for circuit breaker behavior.

CircuitBreakerRegistry

Registry for managing multiple circuit breakers by name. Provides a centralized way to create and access circuit breakers across the application.

ExponentialBackoffDelayStrategy

A delay strategy that implements exponential backoff with jitter. Delay increases exponentially with each retry attempt to reduce load on failing services.

FixedDelayStrategy

A delay strategy that uses a fixed delay between all retry attempts.

PollyResilienceOptions

Configuration options for Polly resilience pipelines.

PollyResiliencePipelineFactory

Factory for creating Polly resilience pipelines with circuit breaker, retry, and timeout policies. Provides pre-configured pipelines for common infrastructure components like PostgreSQL and Redis.

RateLimitOptions

Configuration options for rate limiting behavior.

ResilienceTelemetryListener

Telemetry listener for resilience pipeline events. Collects metrics for monitoring circuit breaker state, retry attempts, and timeouts.

RetryOptions

Configuration options for retry behavior.

RetryPolicy

Default implementation of retry policy that supports configurable retry strategies, delay strategies, and retry conditions.

ServiceCollectionExtensions

Extension methods for registering resilience services in the dependency injection container.

TokenBucketRateLimiter

Token bucket algorithm implementation of rate limiter. Maintains a bucket of tokens for each key, refilling at a steady rate and consuming tokens for requests.

Interfaces

ICircuitBreaker

Provides circuit breaker functionality to prevent cascading failures in distributed systems. Monitors operation failures and temporarily blocks requests when failure thresholds are exceeded.

IDelayStrategy

Defines a strategy for calculating delays between retry attempts.

IRateLimiter

Provides rate limiting functionality to control the rate of requests or operations. Helps prevent system overload and ensures fair resource usage.

IRetryPolicy

Provides retry functionality for operations that may fail transiently. Automatically retries failed operations based on configurable policies.
