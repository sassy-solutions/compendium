Represents a DDD-style orchestration saga ("Process Manager" in Vaughn Vernon's terminology) — a stateful coordinator that groups operations across multiple aggregates within a bounded context, drives them through explicit steps, and performs compensation on failure.

public interface IProcessManager : ISagaInstance

Remarks

Choose a Process Manager over IEventChoreography when:

  • The workflow is owned by a single bounded context.
  • You need centralized visibility of progress (steps, status).
  • The compensation logic is non-trivial and benefits from a single owner.

Properties

Gets the read-only list of steps that comprise this process manager. Steps execute in Order; compensation runs in reverse.
