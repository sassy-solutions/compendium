Drives the lifecycle of process managers: starts new instances, dispatches steps, and triggers compensation on failure.

CompensateAsync(Guid, CancellationToken)

Runs compensation for a process manager: each completed step is undone in reverse order via CompensateAsync(IProcessManager, SagaStep, CancellationToken). On full success the saga reaches Compensated; on partial failure it is left in Compensating.

ExecuteStepAsync(Guid, string, CancellationToken)

Executes a single named step of a process manager. On failure, the step is marked Failed; the caller is expected to invoke CompensateAsync(Guid, CancellationToken) to roll back. The orchestrator does not auto-compensate, so callers can choose to retry transient failures before triggering compensation.
