Source of tools an agent can call during its loop. Implementations may aggregate tools from multiple providers (e.g. a Nexus MCP-backed registry, an in-process HTTP-tool registry, etc.).

Returns

Task<Result<AgentToolResult>>

A Result<TValue> wrapping AgentToolResult. Tool-internal failures (e.g. an HTTP 404 from the underlying API) should be surfaced as a successful result with IsError = true so the agent can let the model see the error and react. A Failure(Error) here means the registry itself failed (unknown tool, parser error, …) and should abort the agent run.
